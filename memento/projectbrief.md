# Project Brief: TheTrybe Web Automation

## 1. Project Overview
This project is a comprehensive web automation framework designed to test the functionality of "The Trybe" e-commerce website. It uses Selenium WebDriver for browser automation and Cucumber for Behavior-Driven Development (BDD). The primary goal is to create robust, maintainable, and scalable automated tests for critical user flows.

## 2. Core Objectives
- **Automate End-to-End User Journeys:** Cover key flows such as searching for products, adding items to the cart, managing a wishlist, and completing the checkout process.
- **Ensure Code Quality & Maintainability:** Adhere to best practices like the Page Object Model (POM), separation of concerns, and data-driven testing.
- **Enable Behavior-Driven Development (BDD):** Write test scenarios in a human-readable format (Gherkin) that is accessible to both technical and non-technical stakeholders.
- **Provide Clear Reporting:** Integrate with reporting tools like ExtentReports and Allure to provide detailed and actionable test results.

## 3. Scope
The scope of this project includes automating tests for the following areas:
- User Authentication (Login/Logout)
- Product Search and Filtering
- Product Detail Page (PDP) interactions
- Shopping Cart management
- Wishlist functionality
- Checkout and Order Placement
